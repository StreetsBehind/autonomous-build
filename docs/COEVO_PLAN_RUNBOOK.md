# Co-evolution runbook (PLAN-CREATION scope) — smbuild ⇄ autonomous-build

The procedure each **tick** of `coevo_smbuild.sh --scope plan` executes. Scope is **plan
creation only**: `/vision` → `/decompose`, iterated until the decompose verdict is **BLESSED**.
There is **no `/build-batch` and no `/retro`** in this scope — no app code is written, no beads
are built, no merges happen. The worst case of a wrong fix is a regenerated plan you review.

A tick is one headless `claude --print`: assess state, advance one step, append a journal
entry, emit exactly one `COEVO_STATUS=` line.

## Goal & current context (read this first)

Drive smbuild's plan to **BLESSED** and, along the way, surface any *residual* workflow defect
the plan-creation pipeline still has. This is primarily a **regression test of the already-shipped
fix `autonomous-build-3fr.1`** ("/vision picks generic formulas, not stack-native variants").

Established facts (do not re-litigate):
- The stack-native formulas **already exist** (`app-skeleton-rust-cargo`, `crud-feature-rust`,
  `grpc-tonic-service`, `audit-chain-rust`, `tenant-boot-rust`, `oidc-client-rust`,
  `otel-bootstrap-rust`, `terraform-aws-baseline`, plus shared `migration`). **Do NOT draft a new
  formula for a capability that already has a native variant** — that is duplicate work.
- The selection fix is **closed**: `skills/vision/SKILL.md` step 6 + the lock cross-check, and
  `decompose.spec.md` Phase-3, now encode "off-enum value on a generic formula = wrong-formula
  signal → switch to the stack-native variant" (per `docs/DEFAULT_STACK.md`).
- smbuild has been **reset to a clean plan-creation slate** (only `vision.md` + `tenets.md`;
  `plan.md`/`plan.lock.json`/ADRs/bead-DAG removed). **The loop owns smbuild** for this run.

So smbuild's prior NEEDS-FIX was a **stale plan** (generated by the old /vision). Regenerating
it with the fixed /vision should now bind native variants and pour cleanly.

## The two lanes

- **App lane (primary, almost every tick)** — `cd smbuild`; run `/vision` then `/decompose`;
  read the verdict. Idempotent regeneration is fine — the loop owns smbuild.
- **Meta lane (rare)** — `cd autonomous-build`; only if a **genuinely NEW** residual defect blocks
  BLESSED that the closed 3fr.1 fix does not cover. Route every fix through `/loop /build-next`
  (the post-build gate). Honor the meta-quiescence pre-check below (parallel sessions).

## Stage detection (every tick, in order)

| Observed state | Tick action |
|---|---|
| smbuild has **no `plan.lock.json`** (fresh slate, or after a clear) | App: run `/vision` → produces plan.md/plan.lock.json/tenets.md. Report. |
| `plan.lock.json` present, `incomplete: true` / blocking `openQuestions`, **and `vision.md` is NOT newer than the lock** | `/escalate` (vision flagged a product/secrets decision) → `sleep-poll`. Do not invent answers. |
| `plan.lock.json` present + `incomplete: true`, **but `vision.md` mtime is newer than the lock** (a human answered the openQuestions and edited the brief) | App: input changed → re-run `/vision` (do NOT escalate the stale lock; the regen should clear the resolved questions). |
| `plan.lock.json` present, complete; no fresh `decomposeReport.md` | App: clear any stale poured beads, then run `/decompose`. Report verdict. |
| `decomposeReport.md` verdict **BLESSED** | `stop-done`. Plan creation succeeded; leave the plan for review, notify. |
| verdict **NEEDS-FIX**, and `/vision` has **not** been re-run this run | App: re-run `/vision` (the fix should now pick native variants), then re-`/decompose`. |
| verdict **NEEDS-FIX** persists **after** a fixed-/vision regen → the cause is a **NEW** defect | Diagnose: mechanical workflow bug → **meta lane** (capture + fix via gate); product/secrets/architecture → `/escalate` → `sleep-poll`. |
| nothing actionable | `sleep-poll`. |

Cheap checks: `ls smbuild/plan.lock.json`; `python3 -c "import json;print(json.load(open('smbuild/plan.lock.json')).get('incomplete'))"`; `grep -m1 'Verdict:' smbuild/decomposeReport.md`. Track in the journal whether `/vision` already ran this run (so you don't loop it forever).

## Autonomy policy (this scope)

- **App lane is fully autonomous** — re-run `/vision`/`/decompose` freely; regenerate plan
  artifacts in the working tree. The loop owns smbuild and nothing is built, so the blast radius
  is a regenerated plan.
- **Meta lane only for NEW residual defects**, and every fix goes through a bead + the
  `/build-next` gate — never a blind edit. A "new defect" is one the closed 3fr.1 fix does not
  already address (verify against `skills/vision/SKILL.md` step 6 + `DEFAULT_STACK.md` before
  assuming a defect is new). **Do not draft a formula that already exists.**
- **Escalate (do not guess, T1):** any blocking `openQuestions` in the lock; secrets/auth
  semantics the plan didn't specify; a true product/vision gap; a capability that genuinely has
  **no** native variant and needs a brand-new formula (that's a design decision — file the bead,
  escalate, do not self-author overnight).

## Meta-quiescence pre-check (parallel sessions on the shared checkout)

Before ANY meta edit: `git -C autonomous-build status --porcelain`. If a file you intend to edit
is already dirty (changes you didn't make this tick), another session is on it — skip the meta
lane this tick and `sleep-poll`. Take `autonomous-build/.coevo-meta.lock` before editing, remove
it after; if it exists and is fresh (<1h), `sleep-poll`. Do not pick up beads `in_progress` or
assigned to a human.

## Commit rules

- **Stage explicit paths; commit explicit paths (`git commit -- <paths>`). NEVER `git add -A`.**
  The user runs parallel Claude sessions on this checkout.
- **smbuild (app lane):** the loop owns it. Do NOT commit on every iteration — regenerate in the
  working tree. When you reach **BLESSED**, you MAY commit the final plan
  (`git -C smbuild commit -- plan.md plan.lock.json tenets.md`) so the blessed state is captured;
  otherwise leave it clean for review. Do not commit `target/` or `.beads/` churn.
- **autonomous-build (meta lane):** commit reference the bead `(bd: autonomous-build-<id>)`, then
  reinstall (`bash install.sh`) so the runtime picks up the change before the next `/vision`.

## Re-pour idempotency

Before re-running `/decompose` after a `/vision` regen, clear the stale poured molecule from the
prior iteration so beads aren't duplicated (`bd list` to see what's there; remove the prior
molecule's issues, or rely on `/decompose`'s own re-pour handling if present — verify it doesn't
double-pour). smbuild's beads are currently empty, so the first decompose is clean.

## Journal

Append per tick to `.orchestrator-logs/coevo-smbuild/journal.md`:
```
## tick <N> — <UTC ts> — lane=<app|meta> step=<vision|decompose|meta-fix|escalate>
- state: plan.lock=<present|absent> incomplete=<...> verdict=<BLESSED|NEEDS-FIX|none>; vision-ran-this-run=<yes|no>
- action / result: <...>
- residual defects captured / escalated: <ids or "none">
- STATUS: <the COEVO_STATUS line>
```
Read it first every tick — it's the only memory across ticks, and it's where you record whether
`/vision` already ran (the anti-infinite-loop signal).

## STATUS contract (the line bash parses)

End the final message with EXACTLY one line, nothing after:
```
COEVO_STATUS=<phase>|<result>|<next>
```
- `<next>` ∈:
  - `continue` — more plan-creation progress available now (driver loops at once).
  - `sleep-poll` — at an escalation/openQuestions gate or nothing actionable (driver polls).
  - `stop-done` — **decompose verdict == BLESSED** (plan creation succeeded).
  - `stop-blocked` — a NEEDS-FIX cause that needs a human design/product/secrets call and cannot
    be cleared by re-vision or a safe meta-fix (driver notifies and exits).

## Notifications

Genuine gate (blocking openQuestions, escalate-class residual defect, BLESSED reached) → invoke
`/escalate` for the consolidated `PushNotification`. The driver emits an `openclaw` event + Discord
check-in each tick — don't duplicate that.

## Hard guardrails — do NOT

- Do **not** run `/build-batch` or `/retro` — out of scope.
- Do **not** draft a formula for a capability that already has a native variant (check the
  `DEFAULT_STACK.md` map first).
- Do **not** loop `/vision` endlessly — if a regen with the fixed /vision doesn't clear a NEEDS-FIX
  cause, that cause is NEW: meta-fix it or escalate; don't re-vision a third time hoping it changes.
- Do **not** invent answers to blocking `openQuestions` (product/secrets) — escalate.
- Do **not** `git add -A`; commit explicit paths only; honor the meta-quiescence pre-check.
